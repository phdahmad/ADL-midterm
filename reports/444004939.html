Question: What is the primary function of "Positional Encoding" in Transformers?

Student Answer:

Question 11 — Essay — (3.5 / 4)

Question: Explain what Word Embedding and Positional Encoding are, and how they are combined to form the input of a Transformer model.

Student Answer:
Model Answer:
Word Embedding: Word Embedding transforms each discrete token into a dense numerical vector that captures its semantic meaning and lexical relationships in a high-dimensional space. Similar words end up close to each other in this space (e.g., 'king' and 'queen' are nearby).
Positional Encoding: Positional Encoding is a mathematical vector (typically computed using sine and cosine functions of different frequencies) that represents the absolute and/or relative position of each token in the sequence. It is needed because Transformers process tokens in parallel and have no inherent sense of order.
Combination: The two vectors (Word Embedding and Positional Encoding) have the same dimension and are combined via element-wise addition: Input = Word Embedding + Positional Encoding. The result is a single vector per token that carries both meaning and position, fed into the first Encoder/Decoder layer.
Analysis & Score Justification:
Excellent and thoughtful answer. (1) Word Embedding clearly defined (context/meaning of a word but not position). (2) Positional Encoding clearly defined with reason (word context relying on location). (3) Combination explanation is good (combining gives both meaning and positioning for context) with a sophisticated note about scaling. (4) No concrete example. The instructor wrote in Arabic 'Excellent, excellent — you impressed me'.

Question 12 — Essay — (4.0 / 5)

Question: Compare Encoder-Only and Decoder-Only Transformers. Explain the type of attention used in each, their primary goals, and why ChatGPT is considered a Generative Model.

Student Answer:
Model Answer:
Encoder-Only vs Decoder-Only:
• Attention:
- Encoder-Only: Bi-directional Self-Attention (Full Attention) — every token sees all other tokens in both directions
- Decoder-Only: Masked / Causal Self-Attention — each token sees only previous tokens; future tokens are masked
• Goal:
- Encoder-Only: Understanding and representation — produces rich Context-Aware Embeddings
- Decoder-Only: Text generation — predicts the next token autoregressively based on prior tokens
• Example tasks/models:
- Encoder-Only: BERT, RoBERTa — Sentiment Analysis, NER, Text Classification, extractive QA
- Decoder-Only: GPT family, LLaMA, Claude, Gemini — Text generation, Dialogue/Chatbots, Code completion
Why ChatGPT is generative: ChatGPT is built on a Decoder-Only auto-regressive architecture. Instead of just analyzing or classifying existing text, it computes the conditional probability of the next token given all prior tokens, allowing it to produce and construct entirely new, coherent textual content in response to user prompts. This generative (rather than discriminative) nature is what defines it as a Generative Model.

Mixed quality. (1) Attention types correctly distinguished. (2) Encoder goal 'Predict previous and next word' is INCORRECT — encoder-only goal is understanding/representation, not prediction. Decoder goal 'Predict next word' is correct. (3) Examples 'BERT' and 'GPT' are model names, not tasks — the instructor marked X on both. However, they do show conceptual understanding of which architecture each model uses. (4) ChatGPT-generative explanation is clear and links decoder-only structure to next-token prediction.
